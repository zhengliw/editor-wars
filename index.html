Schreib.

Um verschiedene Arten von Text und Code schreiben und bearbeiten zu können, braucht man einen Texteditor. Und dafür gibt es viele Möglichkeiten.

Diese Seite hilft dir dabei, einen Texteditor auszuwählen.

Was einen Editor ausmacht (und was nicht)

Nicht alle Programme, die Textdateien öffnen und bearbeiten können, lassen sich als einen Texteditor einstufen. Man unterscheidet zwischen Texteditoren (Vim, Code...) und Textverarbeitungsprogrammen (Word, Writer...). Texteditoren arbeiten mit reinem Text (plain text) - die Dateien enthalten nur Text oder Code und können mit jedem anderen Texteditor geöffnet werden. Man sieht alles, was in der Datei enthalten ist.

Textverarbeitungsprogramme speichern zwar auch das Geschriebene in Dateien, jedoch in eigenen Dateiformaten, wo sie neben Text noch strukturielle Daten wie Layoutparameter oder Dokumenteinstellungen hinterlegen. Sie eignen sich dadurch besser für Dokumente, die ein bestimmtes Layout oder eingebettete Elemente wie Bilder benötigen.

Ganz alte Editors

Auch wenn sie etwas ungewöhnlich und vielleicht etwas unpraktisch aussehen, bieten viele Editors, die im Terminal laufen, einen ganz eigenen Workflow.

(Neo)Vim bezeichnet sich zum Beispiel als einen multi-modalen Texteditor. Im Normal Mode lassen sich Texte in beliebigen Mengen löschen, ändern oder kopieren, im Insert Mode kann man Texte einfügen und im Visual Mode kann man visuell (wie der Name schon besagt) Textblöcke auswählen und mit denen verschiedene Aktionen ausführen.

Neue Mitspieler

Typisch für heutige (moderne?) Entwicklung-Workflows sind jedoch grafische Editors, ausgestattet mit einer GUI (graphical user interface). Oft kommen solche Editors mit vielen Features out-of-the-box vorinstalliert und sind mit einer intuitiven Bedienoberfläche ausgestattet.
